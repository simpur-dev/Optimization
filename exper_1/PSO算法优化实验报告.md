## 三、算法原理

### 3.1 粒子群优化算法简介

粒子群优化算法（Particle Swarm Optimization, PSO）是一种基于群体智能的优化算法，由Kennedy和Eberhart于1995年提出。算法模拟鸟群觅食行为，通过群体协作寻找最优解。

### 3.2 核心公式

**速度更新公式：**
```
v_i(t+1) = w × v_i(t) + c1 × r1 × (pbest_i - x_i(t)) + c2 × r2 × (gbest - x_i(t))
```

**位置更新公式：**
```
x_i(t+1) = x_i(t) + v_i(t+1)
```

其中：
- `w`：惯性权重，控制粒子保持原有运动趋势的程度
- `c1`：个体学习因子，控制粒子向自身历史最优位置学习的程度
- `c2`：全局学习因子，控制粒子向全局最优位置学习的程度
- `r1, r2`：[0,1]区间内的随机数
- `pbest_i`：粒子i的历史最优位置
- `gbest`：群体全局最优位置

### 3.3 惯性权重策略

本实验采用**线性递减惯性权重**策略：
```
w = w_max - (w_max - w_min) × iter / max_iter
```

- **前期**（w较大）：增强全局搜索能力，避免陷入局部最优
- **后期**（w较小）：增强局部搜索能力，提高收敛精度

## 四、目标函数介绍

### 4.1 Griewank函数

**数学表达式：**
```
f(x1, x2) = 1 + (x1² + x2²)/4000 - cos(x1/√1) × cos(x2/√2)
```

**特性：**
- 搜索范围：[-600, 600]
- 理论全局最小值：f(0, 0) = 0
- 函数特点：具有大量局部极小值，全局最优位于原点
- 难度：中等，适合测试算法的全局搜索能力

**代码实现：**
```cpp
double Griewank(double x1, double x2)
{
    double sum_sq = x1 * x1 + x2 * x2;
    double prod_cos = cos(x1 / sqrt(1.0)) * cos(x2 / sqrt(2.0));
    return 1.0 + sum_sq / 4000.0 - prod_cos;
}
```

### 4.2 Rastrigin函数

**数学表达式：**
```
f(x1, x2) = 20 + (x1² + x2²) - 10 × [cos(2πx1) + cos(2πx2)]
```

**特性：**
- 搜索范围：[-5.12, 5.12]
- 理论全局最小值：f(0, 0) = 0
- 函数特点：高度多模态，存在大量均匀分布的局部极小值
- 难度：高，是测试优化算法性能的经典函数

**代码实现：**
```cpp
double Rastrigin(double x1, double x2)
{
    double sum_sq = x1 * x1 + x2 * x2;
    double sum_cos = cos(2 * M_PI * x1) + cos(2 * M_PI * x2);
    return 20.0 + sum_sq - 10.0 * sum_cos;
}
```

## 五、算法实现

### 5.1 数据结构设计

#### 粒子结构体
```cpp
struct Particle
{
    double x1, x2;               // 粒子位置（二维）
    double v1, v2;               // 粒子速度
    double pbest_x1, pbest_x2;   // 个体历史最优位置pbest
    double pbest_fit;             // 个体历史最优适应度
};
```

**设计说明：**
- 采用结构体封装粒子的所有属性
- 支持二维优化问题
- 存储个体历史最优信息，实现记忆功能

#### 全局最优变量
```cpp
double gbest_x1, gbest_x2;    // 全局最优位置
double gbest_fit = 1e10;       // 全局最优适应度
```

### 5.2 算法参数设置

| 参数名称 | 参数值 | 说明 |
|---------|--------|------|
| pop_size | 50 | 种群规模（粒子数量） |
| max_iter | 1000 | 最大迭代次数 |
| c1 | 2.0 | 个体学习因子 |
| c2 | 2.0 | 全局学习因子 |
| w_max | 0.9 | 最大惯性权重 |
| w_min | 0.4 | 最小惯性权重 |
| v_max | 0.2×(x_max-x_min) | 速度上限 |

**参数选择依据：**
- **种群规模50**：平衡计算效率和搜索能力
- **学习因子c1=c2=2.0**：经典设置，平衡个体和群体学习
- **惯性权重[0.4, 0.9]**：线性递减策略，前期探索后期收敛
- **速度限制20%搜索范围**：防止粒子飞出搜索空间

### 5.3 核心算法流程

#### 初始化阶段
```cpp
// 随机初始化位置
pop[i].x1 = x_min + (x_max - x_min) * (double)rand() / RAND_MAX;
pop[i].x2 = x_min + (x_max - x_min) * (double)rand() / RAND_MAX;

// 随机初始化速度
pop[i].v1 = v_min + (v_max - v_min) * (double)rand() / RAND_MAX;
pop[i].v2 = v_min + (v_max - v_min) * (double)rand() / RAND_MAX;

// 计算初始适应度并初始化个体最优
double init_fit = func(pop[i].x1, pop[i].x2);
pop[i].pbest_x1 = pop[i].x1;
pop[i].pbest_x2 = pop[i].x2;
pop[i].pbest_fit = init_fit;

// 更新全局最优
if (pop[i].pbest_fit < gbest_fit)
{
    gbest_fit = pop[i].pbest_fit;
    gbest_x1 = pop[i].pbest_x1;
    gbest_x2 = pop[i].pbest_x2;
}
```

#### 迭代优化阶段
```cpp
// 惯性权重线性递减
double w = w_max - (w_max - w_min) * (double)iter / max_iter;

// 速度更新（三部分：惯性 + 个体学习 + 全局学习）
pop[i].v1 = w * pop[i].v1 
          + c1 * r1 * (pop[i].pbest_x1 - pop[i].x1) 
          + c2 * r2 * (gbest_x1 - pop[i].x1);

// 速度越界限制
pop[i].v1 = (pop[i].v1 > v_max) ? v_max : pop[i].v1;
pop[i].v1 = (pop[i].v1 < v_min) ? v_min : pop[i].v1;

// 位置更新
pop[i].x1 += pop[i].v1;

// 位置越界限制
pop[i].x1 = (pop[i].x1 > x_max) ? x_max : pop[i].x1;
pop[i].x1 = (pop[i].x1 < x_min) ? x_min : pop[i].x1;

// 更新个体最优
double current_fit = func(pop[i].x1, pop[i].x2);
if (current_fit < pop[i].pbest_fit)
{
    pop[i].pbest_x1 = pop[i].x1;
    pop[i].pbest_x2 = pop[i].x2;
    pop[i].pbest_fit = current_fit;
}

// 更新全局最优
if (pop[i].pbest_fit < gbest_fit)
{
    gbest_fit = pop[i].pbest_fit;
    gbest_x1 = pop[i].pbest_x1;
    gbest_x2 = pop[i].pbest_x2;
}
```

### 5.4 模块化设计

本实验采用**函数指针**实现算法与目标函数的解耦：

```cpp
void runPSO(double (*func)(double, double), double x_min, double x_max, const string& func_name)
```

**优势：**
- 算法逻辑与目标函数分离
- 支持多种目标函数，易于扩展
- 代码复用性高

**调用方式：**
```cpp
runPSO(Griewank, -600.0, 600.0, "Griewank");
runPSO(Rastrigin, -5.12, 5.12, "Rastrigin");
```

## 七、结果分析与讨论

### 7.1 算法性能分析

#### 优势
1. **全局搜索能力强**：成功找到两个多模态函数的全局最优解
2. **收敛速度快**：在500次迭代内达到最优
3. **精度高**：最优解误差在10⁻⁹~10⁻¹⁰级别
4. **鲁棒性好**：对不同特性的函数都能有效优化

#### 关键因素
1. **惯性权重递减策略**：平衡了全局搜索和局部搜索
2. **合理的参数设置**：学习因子c1=c2=2.0，种群规模50
3. **边界处理机制**：防止粒子飞出搜索空间
4. **双重最优更新**：个体最优和全局最优协同作用

### 7.2 两个函数的优化难度对比

| 特性 | Griewank函数 | Rastrigin函数 |
|-----|-------------|--------------|
| 搜索范围 | [-600, 600] | [-5.12, 5.12] |
| 局部极小值数量 | 较多 | 极多 |
| 收敛迭代次数 | ~500次 | ~400次 |
| 优化难度 | 中等 | 高 |
| 实际表现 | 成功 | 成功 |

**分析：**
- Rastrigin函数虽然局部极小值更多，但搜索范围更小
- Griewank函数搜索范围大，需要更多迭代才能收敛
- 两个函数都成功优化，说明算法具有良好的适应性

### 7.3 参数敏感性分析

#### 惯性权重的影响
- **w过大**（>0.9）：全局搜索能力强，但收敛慢
- **w过小**（<0.4）：收敛快，但易陷入局部最优
- **线性递减策略**：兼顾全局和局部搜索

#### 学习因子的影响
- **c1=c2=2.0**：经典设置，平衡个体和群体学习
- **c1>c2**：强调个体经验，适合多峰函数
- **c1<c2**：强调群体经验，适合单峰函数

#### 种群规模的影响
- **规模过小**（<20）：搜索能力不足，易陷入局部最优
- **规模过大**（>100）：计算开销大，收敛慢
- **规模50**：平衡搜索能力和计算效率

## 八、实验结论

### 8.1 主要结论

1. **算法有效性**：PSO算法成功求解了Griewank和Rastrigin两个经典多模态优化函数，找到了全局最优解。

2. **收敛性能**：算法在500次迭代内收敛，收敛速度快，优化精度高（误差在10⁻⁹级别）。

3. **参数合理性**：采用的参数设置（种群规模50，学习因子2.0，惯性权重[0.4, 0.9]）对两个函数都表现出良好的优化性能。

4. **策略有效性**：线性递减惯性权重策略有效平衡了全局搜索和局部搜索能力。

5. **代码设计**：模块化设计（函数指针）使算法具有良好的可扩展性和复用性。

### 8.2 算法优势

1. **原理简单**：算法思想直观，易于理解和实现
2. **参数少**：主要参数只有种群规模、学习因子和惯性权重
3. **收敛快**：相比遗传算法等，PSO收敛速度更快
4. **全局搜索能力强**：能有效跳出局部最优

### 8.3 改进建议

1. **自适应参数**：可以采用自适应学习因子，根据搜索状态动态调整
2. **多种群策略**：使用多个子种群并行搜索，提高全局搜索能力
3. **混合算法**：结合局部搜索算法（如梯度下降），提高收敛精度
4. **约束处理**：增加约束处理机制，支持约束优化问题
